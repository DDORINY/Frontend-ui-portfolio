v2 Changes
  • Spacing / Radius 토큰을 더 촘촘하게 확장(레이아웃 미세 조정)
  • Theme Variant(예: Ocean / Grape) 구조 제안
  • Dark Mode는 토큰만 교체하는 전략으로 설계

v2에서는 theme.css는 토큰만 유지하면서, 변형 테마는 별도 파일로 분리하는 방식이 깔끔함. (예: theme-ocean.css, theme-grape.css)

/* theme-grape.css (예시) */ :root{ --brand: #7c3aed; --brand-600: #6d28d9; --brand-50: #f5f3ff; }

4) Dark Mode Strategy

다크모드는 “컴포넌트 코드를 바꾸는 게 아니라” 토큰만 바꾸는 게 핵심. v2에서는 아래처럼 data-theme="dark"로 토큰만 오버라이드 권장.
/* theme-dark.css (예시) */ :root[data-theme="dark"]{ --bg: #0b1220; --surface: rgba(255,255,255,.06); --text: #e5e7eb; --muted: rgba(229,231,235,.72); --border: rgba(229,231,235,.12); --border-strong: rgba(229,231,235,.20); --shadow: 0 12px 30px rgba(0,0,0,.35); --shadow-lg: 0 18px 40px rgba(0,0,0,.45); --brand-50: rgba(37,99,235,.12); }
토글 예시(개념)
/* JS 예시 */ document.documentElement.dataset.theme = document.documentElement.dataset.theme === "dark" ? "" : "dark";
UI Kit 문서에선 JS를 꼭 넣을 필요는 없고, “전략”을 문서화만 해도 충분.

Usage

<!-- 기본 --> <link rel="stylesheet" href="assets/css/theme.css"> <link rel="stylesheet" href="assets/css/components.css"> <!-- 변형 테마(선택) --> <link rel="stylesheet" href="assets/css/theme-grape.css"> <!-- 다크 모드(선택) --> <link rel="stylesheet" href="assets/css/theme-dark.css">

Guidelines

  • theme.css는 “토큰만” 유지(컴포넌트 스타일 넣지 않기)
  • Variant/Dark는 별도 파일로 분리해서 “선택 로딩” 가능하게
  • 프로젝트 규모 커지면 토큰 표를 마크다운/스프레드시트로 관리해도 좋음
